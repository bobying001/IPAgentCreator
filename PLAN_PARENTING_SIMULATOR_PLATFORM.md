# 项目规划：NLP 亲子关系实战模拟器 (Parenting Dojo)

## 1. 产品愿景 (Vision)
一个基于“费曼技巧”与“角色扮演”的深度学习平台。让父母不再只是“听”道理，而是通过**模拟最坏的孩子**，亲身体验**最好的父母**（李中莹数字分身）是如何化解冲突的。
*Slogan: "Be the Child, Learn to be the Parent."*

## 2. 用户交互流程 (User Journey)

### Step 1: 剧本设定 (Scenario Setup)
用户（父母）配置他们面临的真实难题：
*   **孩子年龄**: 14岁
*   **争吵场景**: 沉迷手机，不做作业。
*   **孩子性格**: 叛逆，软硬不吃。

### Step 2: 角色反转 (Role Reversal)
*   **用户扮演**: **孩子**（User 模拟自家孩子的语气说话）。
*   **AI 扮演**: **李中莹老师**（作为理想父母的化身）。

### Step 3: 对话实战 (The Sparring)
*   **User (孩子)**: “烦死了！读书有什么用？我不读了！”
*   **AI (李中莹版父母)**: “看到你这么烦躁，我感觉到你其实很想把事情做好，只是现在遇到了一些困难，是吗？”（运用：先跟后带 + 肯定正面动机）

### Step 4: 实时旁白 (Live Commentary) - *核心差异点*
在聊天界面的右侧（或气泡下方），显示**【专家旁白】**：
> *AI 解析*: 这里没有直接反驳（说教型），而是先接受了孩子的情绪（EQ处理），并尝试找出行为背后的正面动机（想做好单遇到困难）。这能降低孩子的防御机制。

## 3. 核心功能模块

### 3.1 双模态 AI 引擎
*   **Actor Agent (演员智能体)**:
    *   加载 `NLP_Protocol` 和所有 `Atomic.md`。
    *   任务：生成符合李中莹 NLP 逻辑的、口语化的回复。
*   **Tutor Agent (导师智能体)**:
    *   任务：监控 Actor 的回复，分析其中运用了哪个 `Atomic.md` 里的知识点（如：[NLP_EXEC_NOTE_005]），并生成教学注解。

### 3.2 知识图谱挂载
*   当注解中提到“正面动机”时，支持点击弹出我们萃取好的 `NLP_Protocol.md` 中的理论定义。

### 3.3 “抄作业”模式 (Take-away)
*   对话结束后，一键生成“实战复盘报告”。
*   将刚才 AI 说的金句，转化为用户明天可以直接对自己孩子说的话术清单。

## 4. MVP 实施路线图

### Phase 1: 纯文本 MVP
*   **Form**: 一个简单的 Chat 界面。
*   **Logic**:
    *   Prompt 中设定 User = Child, System = Patent (Li Zhongying style).
    *   System 每次回复 JSON 格式：`{ "reply": "...", "analysis": "..." }`
    *   前端解析 JSON，分别渲染“对话气泡”和“旁白卡片”。

### Phase 2: 场景预设库
*   内置 10 个最典型的中国家庭冲突剧本（如：手机成瘾、早恋摊牌、厌学拒考）。
*   用户无需配置，直接点击“开始挑战”。

### Phase 3: 语音交互
*   支持语音输入（模拟真实吵架语速）。
*   AI 语音回复（通过 TTS 传递温和而坚定的语气，李中莹语调）。

## 5. 为什么这个更具粘性？
*   **痛点解决**: 父母看书学会了道理，但张嘴就忘。这个平台提供了“话术示范”。
*   **共情体验**: 父母在扮演孩子的过程中，会第一次真正体会到“被这样的父母（模拟的完美父母）对待”是多么舒服，从而产生深刻的内驱力去改变自己。

## 6. 技术栈建议
*   **Backend**: Python (FastAPI) + Google Gemini Pro (处理角色扮演逻辑优势明显)。
*   **Frontend**: Next.js / React (实现左右分栏的交互界面)。
*   **Database**: 存储用户的实战记录，用于生成个性化建议。
